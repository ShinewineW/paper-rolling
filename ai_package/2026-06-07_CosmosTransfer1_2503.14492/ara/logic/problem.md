# Problem Specification

## Observations

### C1: CG 模拟器与真实世界之间存在显著的合成-真实域差（Sim2Real gap），
- **Statement**: CG 模拟器与真实世界之间存在显著的合成-真实域差（Sim2Real gap），直接使用合成数据训练的模型在真实环境中泛化能力不足
- **Evidence**: 论文第 1 节明确指出，可利用模拟器免费提供的深度与分割信号作为世界生成的条件输入，以缓解域差问题
- **Implication**: 需要一种能将合成渲染转化为高保真度现实视频、同时保留场景结构与语义的生成方法

### C2: 单一模态条件控制在控制精度与生成多样性之间存在固有权衡：视觉模糊（Vis）和边缘
- **Statement**: 单一模态条件控制在控制精度与生成多样性之间存在固有权衡：视觉模糊（Vis）和边缘（Edge）提供稠密结构约束但降低多样性，深度和分割提供稀疏约束但对应指标保真度较低
- **Evidence**: Table 1 中，Cosmos-Transfer1-7B [Vis] 的 Diversity-LPIPS 最低（0.19），Cosmos-Transfer1-7B [Seg] 的 Diversity-LPIPS 最高（0.42），印证了这一权衡关系
- **Implication**: 不同任务场景需要不同模态组合，静态的单一模态或全局均等加权均无法满足精细空间控制需求

### C3: Physical AI（机器人、自动驾驶）领域高质量真实场景数据稀缺，安全关键长
- **Statement**: Physical AI（机器人、自动驾驶）领域高质量真实场景数据稀缺，安全关键长尾场景覆盖不足
- **Evidence**: 论文第 5.3 节引用数据扩展律指出大规模高质量数据对机器人研究的关键性；第 5.4 节指出自动驾驶数据分布呈高度长尾特性，安全关键场景难以充分积累
- **Implication**: 需要能够在保持物理合理结构的同时生成多样化视觉变体的世界生成工具，以扩充训练数据

## Gaps

### C4: 现有多模态条件控制方法缺乏对同一视频不同空间区域施加差异化模态权重的能力
- **Statement**: 现有多模态条件控制方法缺乏对同一视频不同空间区域施加差异化模态权重的能力
- **Caused by**: 原始 ControlNet 设计针对单一条件输入，多模态扩展时缺乏时空感知的权重分配机制
- **Existing attempts**: []
- **Why they fail**: 现有 ControlNet 扩展方案通常对全局空间使用统一权重，无法为前景与背景区域分别指定最优模态组合

### C5: 同时训练多个模态控制分支的内存开销巨大，且不同模态所需配对训练数据难以统一获取
- **Statement**: 同时训练多个模态控制分支的内存开销巨大，且不同模态所需配对训练数据难以统一获取
- **Caused by**: 大规模视频扩散模型训练本身对硬件资源要求极高，多模态联合训练更加剧这一挑战
- **Existing attempts**: []
- **Why they fail**: 视频数据本身内存开销极高，多分支联合训练进一步成倍放大显存压力；部分模态（如 HD 地图）的配对数据在通用场景中不可得

## Key Insight
- **Insight**: 将各模态控制分支独立训练，并引入时空控制图在推理时自适应加权融合，可以同时解决内存约束与模态灵活组合两大问题，并实现区域级精细空间控制
- **Derived from**: C4 与 C5 两个 gap 的分析：分治训练解决内存与数据配对问题，推理期时空加权解决空间差异化控制问题
- **Enables**: 在机器人 Sim2Real（前景用 Edge+Vis 保持形态，背景用 Seg 增加多样性）和自动驾驶（HDMap+LiDAR 互补融合）等应用中实现高质量、高多样性的条件世界生成

## Assumptions
- 各控制分支在独立训练后可在推理时线性叠加融合，彼此干扰可忽略不计（分析推断，论文未显式声明此独立性保证）
- 时空控制图可由规则算法（如 SalientObject 算法）或神经网络模块自动生成，无需人工逐帧标注
- Cosmos-Predict1-7B-Video2World 的基础能力（文本对齐、视频时序一致性）可通过 ControlNet 后训练有效迁移到条件控制任务
