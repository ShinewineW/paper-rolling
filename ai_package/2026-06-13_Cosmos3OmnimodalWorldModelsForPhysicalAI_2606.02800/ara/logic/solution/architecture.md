语言、图像、视频、音频、动作原始输入 -> 模态专用编码器投影到统一隐藏空间；非语言模态额外加入可学习模态嵌入 -> 统一打包为前置 AR 子序列与后置 DM 子序列；AR 承载语言与 ViT 视觉理解 token，DM 承载 VAE 视觉生成 token、音频 token、动作 token，且 DM 内部按 clean conditioning tokens 再 noisy diffusion tokens 排列，并按 vision、audio、action 排序 -> Mixture-of-Transformers 双塔解码层处理同一序列；AR token 路由到 reasoner tower，DM token 路由到 generator tower -> AR 使用因果自注意力保持自回归文本与理解能力，DM 使用对 AR+DM 的全双向注意力以接收文本、条件帧、控制、音频、动作上下文 -> 3D MRoPE 与 absolute temporal modulation 为视频、音频、动作分配共享物理时间坐标，并用 AR-DM temporal gap 缓冲文本到生成 token 的位置过渡 -> 训练时 Reasoner 先做多模态 next-token prediction 与 SFT，Generator 继承 Reasoner 权重后用 rectified flow matching 进行图像、视频、音频预训练，再通过 mid-training 加入 action 与 transfer，最后按 T2I、I2V、Policy-DROID 等目标后训练 -> 推理时语言用 next-token prediction，其他模态通过迭代去噪生成；policy 推理可跳过视频 latent 解码，仅输出动作以降低延迟。
