# Problem Specification

## Observations

### O1: 大型语言模型在编程和数学任务中展现出卓越的长链式推理能力，但难以将所掌握的知识与
- **Statement**: 大型语言模型在编程和数学任务中展现出卓越的长链式推理能力，但难以将所掌握的知识与真实物理世界的动态交互相连接。
- **Evidence**: 论文引用 OpenAI o1 和 DeepSeek-R1 在数学/编程领域的突破，同时指出这些模型「often struggle to establish connections between that knowledge and real-world interactions and dynamics」。
- **Implication**: 现有推理增强方法对物理世界理解的迁移性不足，需要专门针对物理AI的训练范式。

### O2: 当前视觉语言模型缺乏系统化的物理常识与具身推理能力，无法有效支撑机器人操作、人形
- **Statement**: 当前视觉语言模型缺乏系统化的物理常识与具身推理能力，无法有效支撑机器人操作、人形机器人和自动驾驶等物理AI应用。
- **Evidence**: Table 7 和 Table 8 显示，Qwen2.5-VL-7B 在物理常识基准平均准确率仅为47.4，在具身推理基准平均准确率仅为50.8，远低于本文模型。
- **Implication**: 通用多模态预训练不足以赋予模型物理世界的操作性理解，必须引入领域专用数据与训练目标。

### O3: 现有VLM在基础直觉物理任务（时间箭头判断、空间拼图复原、物体永久性推断）上表现
- **Statement**: 现有VLM在基础直觉物理任务（时间箭头判断、空间拼图复原、物体永久性推断）上表现接近随机猜测，说明通用基准无法衡量物理理解深度。
- **Evidence**: Table 10 显示，Gemini 2.0 Flash 在时间箭头任务准确率为50.0（与随机猜测持平），Qwen2.5-VL-7B 在空间拼图任务准确率仅27.2（随机猜测为25.0）。
- **Implication**: 需要专门为物理AI设计的基准测试，而非依赖 MMMU 等通用评估来衡量物理推理能力。

## Gaps

### G1: 缺乏适用于物理AI的可验证奖励设计框架，导致强化学习后训练难以应用于物理常识和具
- **Statement**: 缺乏适用于物理AI的可验证奖励设计框架，导致强化学习后训练难以应用于物理常识和具身推理任务。
- **Caused by**: O1
- **Existing attempts**: ['将数值计算式奖励直接应用于描述性推理答案（不适用）']
- **Why they fail**: 数学/编程领域答案格式明确、可精确比对，而物理常识和具身推理通常需要开放式回答，奖励信号难以自动化核验。

### G2: 缺乏系统化的Physical AI能力本体和专用评估基准，无法量化衡量模型的物理
- **Statement**: 缺乏系统化的Physical AI能力本体和专用评估基准，无法量化衡量模型的物理世界理解进展。
- **Caused by**: O2, O3
- **Existing attempts**: ['以MMMU等通用基准代替物理专项评估（揭示了能力盲区但未解决）']
- **Why they fail**: 现有通用视觉-语言基准（如MMMU）虽覆盖多学科知识，但未专门考察物理世界交互中所需的空间、时间和力学推理能力。

## Key Insight
- **Insight**: 将物理常识和具身推理样本统一转化为单选题（MCQ）格式，使得答案核验可通过字符串匹配实现，从而可以设计基于规则的可验证奖励，将GRPO强化学习后训练系统性地扩展到物理AI推理领域。
- **Derived from**: G1
- **Enables**: 以简单的规则奖励（准确性奖励 + 格式奖励）驱动模型在物理常识、具身推理和直觉物理三类任务上的自我提升，无需人工标注额外奖励信号。

## Assumptions
- 视频输入足以表征物理世界观测，是「处理复杂感知输入」能力的代表性形式。
- MCQ格式能够充分覆盖物理常识和具身推理中的关键判断场景，不会因答案格式转化而损失核心推理信息。（分析推断，论文未显式声明）
- 从 DeepSeek-R1 提炼的推理轨迹质量足够高，可作为 SFT 阶段的监督信号而不引入严重噪声。
