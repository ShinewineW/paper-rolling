输入序列 token -> 对每个 token 独立采样噪声等级 -> ForwardDiffuse 得到逐 token 不同噪声的序列 -> 因果序列模型用过去 latent 与当前 noisy token 更新 latent -> 噪声预测头预测当前 token 的噪声 -> 训练期用常规 diffusion 噪声预测损失同时学习所有 token -> 推理期从白噪声序列开始,按二维 scheduling matrix 逐行、逐列去噪 -> 可选地在采样阶段加入 guidance,使未来 token 的梯度经因果依赖回传到较早 token -> 在 sequential decision making 中把 action、reward、next observation 拼为 token,滚动采样 look-ahead plan 并执行首个 action。
