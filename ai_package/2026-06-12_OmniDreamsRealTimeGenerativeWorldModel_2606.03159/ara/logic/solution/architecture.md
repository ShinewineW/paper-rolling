AlpaSim 接收 Alpamayo 1 或用户动作并更新 simulator state -> 将 trajectory、text prompt、首个 chunk 的 first-frame RGB 和 world-scenario map 发送到 stateful OmniDreams server -> world-scenario 渲染器按相机内外参把 HD map、动态 actor cuboid 和 ego trajectory 变成对齐的视频条件 -> 轻量 control branch 用小型 MLP 将结构化状态编码为 control tokens，并与 noisy latent tokens、first-frame latent、text cross-attention 条件拼接 -> causal transformer backbone 使用 causal masking、streaming KV cache 和 local-window temporal attention 生成下一段 latent frames -> OmniDreams-MV 在每个时间步加入 view embedding 与 Cross-View Attention，使多个 camera views 共享几何和运动一致性 -> 少步 denoising 后由 LightTAE 解码为 RGB/JPEG frames -> server 更新 KV cache、renderer state 和 CUDA Graph state，并把帧返回 AlpaSim -> policy 基于新 observation 继续闭环。训练路径是 Cosmos-Predict 2.5 -> RDS AV mid-training -> OmniDreams-MV 的 view embedding 与 cross-view attention 多视角适配 -> world-scenario control post-training -> Diffusion Forcing causal mid-training -> Self Forcing + DMD distillation -> inference-only 的 FlashDreams 优化与多 GPU serving。
