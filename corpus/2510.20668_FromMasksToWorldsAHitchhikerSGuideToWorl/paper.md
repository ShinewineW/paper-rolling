<div class="ltx_abstract">

###### Abstract

This is not a typical survey of world models; it is a guide for those
who want to build worlds. We do not aim to catalog every paper that has
ever mentioned a “world model”. Instead, we follow one clear road: from
early masked models that unified representation learning across
modalities, to unified architectures that share a single paradigm, then
to interactive generative models that close the action-perception loop,
and finally to memory-augmented systems that sustain consistent worlds
over time. We bypass loosely related branches to focus on the core: the
generative heart, the interactive loop, and the memory system. We show
that this is the most promising path towards true world models.

</div>

## <span class="ltx_tag ltx_tag_section">1 </span>Introduction: The Narrow Road to World Models

<div id="S1.p1" class="ltx_para ltx_noindent">

The term *world model* has been used to describe many different ideas:
learned environment simulators for reinforcement learning (Ha &
Schmidhuber, <a href="https://arxiv.org/html/2510.20668v1#bib.bib39"
class="ltx_ref">2018</a>; Hafner et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib40"
class="ltx_ref">2019</a>), agents that integrate learned models with
planning (Schrittwieser et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib97"
class="ltx_ref">2020</a>), and large language models that simulate
entire societies (Park et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib84"
class="ltx_ref">2023</a>). Yet despite hundreds of related works, there
is no clear consensus on how to actually build a true world model. In
this paper, we take a stance: the path is much narrower than it appears.

</div>

<div id="S1.p2" class="ltx_para ltx_noindent">

A true world model is not a monolithic entity but a system synthesized
from three core subsystems: a generative heart that produces world
states, an interactive loop that closes the action-perception cycle in
real time, and a persistent memory system that sustains coherence over
long horizons. The history of the field can be understood as an
evolutionary journey from first mastering these components in isolation
to now integrating them. Most works focus on optimizing narrow tasks and
drift away from the generative, interactive, and persistent nature
required for a true world model.

</div>

<div id="S1.p3" class="ltx_para ltx_noindent">

To make this perspective concrete, we chart the historical evolution of
world models as a sequence of five stages, shown in
Figure <a href="https://arxiv.org/html/2510.20668v1#S1.F1" class="ltx_ref"
title="Figure 1 ‣ 1 Introduction: The Narrow Road to World Models ‣ From Masks to Worlds: A Hitchhiker’s Guide to World Models"><span
class="ltx_text ltx_ref_tag">1</span></a>. It begins with Stage I:
Mask-based Models, which established a universal, token-based
pretraining paradigm across modalities. This foundation enabled Stage
II: Unified Models, where a single architecture learns to process and
generate multiple modalities. The focus then shifts to closing the
interactive loop in Stage III: Interactive Generative Models,
transforming static generators into real-time simulators. To sustain
these simulations over time, Stage IV: Memory and Consistency introduces
mechanisms for durable and coherent state representation.
Table <a href="https://arxiv.org/html/2510.20668v1#S1.T1" class="ltx_ref"
title="Table 1 ‣ 1 Introduction: The Narrow Road to World Models ‣ From Masks to Worlds: A Hitchhiker’s Guide to World Models"><span
class="ltx_text ltx_ref_tag">1</span></a> also summarizes representative
models or methods across the four stages.

This progression culminates in Stage V: True World Models. This stage is

preceding stages into an autonomous whole. At this threshold, models

analyzing each stage’s key innovations and unsolved challenges, this
paper offers a clear and opinionated roadmap from today’s components to
tomorrow’s living worlds.

<div id="S2" class="section ltx_section">

## <span class="ltx_tag ltx_tag_section">2 </span>What is a World Model?

<div id="S2.SS1" class="section ltx_subsection">

### <span class="ltx_tag ltx_tag_subsection">2.1 </span>Historical and Contemporary Perspectives

<div id="S2.SS1.p1" class="ltx_para ltx_noindent">

The concept of a world model originated in reinforcement learning, where
Ha and Schmidhuber (Ha & Schmidhuber,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib39"
class="ltx_ref">2018</a>) first proposed learning a latent dynamics
simulator for agent planning. This control-oriented view was advanced by
systems like Dreamer (Hafner et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib40"
class="ltx_ref">2019</a>), which learned policies purely through latent
imagination, and MuZero (Schrittwieser et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib97"
class="ltx_ref">2020</a>), which integrated tree-based planning with a
learned, abstract model. In parallel, the rise of large-scale generative
modeling broadened this definition. With generative agents (Park et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib84"
class="ltx_ref">2023</a>) and large multimodal systems (Reed et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib93"
class="ltx_ref">2022</a>), the concept evolved from a predictive
simulator for an agent to a rich, generative system capable of an entire
interactive world. This has led to the contemporary view of a “world
simulator”, a term that now informally encompasses three major
paradigms: explicit 3D scene generators (World Labs,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib112"
class="ltx_ref">2024</a>), passive video generators that go beyond
pixels to approximate physical dynamics (Brooks et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib10"
class="ltx_ref">2024</a>), and interactive games and environments for
agents, whether text-based (Niesz & Holland,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib80"
class="ltx_ref">1984</a>) or video-based, as exemplified by the Genie
series (Bruce et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib12"
class="ltx_ref">2024</a>; Parker-Holder et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib85"
class="ltx_ref">2024</a>; Ball et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib6"
class="ltx_ref">2025</a>).

</div>

</div>

<div id="S2.SS2" class="section ltx_subsection">

### <span class="ltx_tag ltx_tag_subsection">2.2 </span>The Anatomy of a True World Model

<div id="S2.SS2.p1" class="ltx_para ltx_noindent">

To bring clarity to these diverse threads, we define a true world model
as the three essential subsystems it must integrate, which, in turn,
enable the core properties that define each stage of our evolutionary
roadmap.
Figure <a href="https://arxiv.org/html/2510.20668v1#S2.F2" class="ltx_ref"
title="Figure 2 ‣ 2.2 The Anatomy of a True World Model ‣ 2 What is a World Model? ‣ From Masks to Worlds: A Hitchhiker’s Guide to World Models"><span
class="ltx_text ltx_ref_tag">2</span></a> presents the high-level
architecture of a true world model, showing how the generative,
interactive, and memory subsystems integrate.

</div>

<figure id="S2.F2" class="ltx_figure">
<img src="/html/2510.20668v1/x2.png" id="S2.F2.g1"
class="ltx_graphics ltx_centering ltx_img_landscape" width="609"
height="342" alt="Refer to caption" />
<figcaption><span class="ltx_tag ltx_tag_figure">Figure 2: </span> The
architecture of a true world model.</figcaption>
</figure>

<div id="S2.SS2.SSS0.Px1" class="section ltx_paragraph">

#### The Generative Heart ($`\mathcal{G}`$).

<div id="S2.SS2.SSS0.Px1.p1" class="ltx_para ltx_noindent">

The foundation of a world model is its generative heart: a learned model
of the world’s dynamics and appearance, formally described by the
generative process $`p_{\theta}`$. It must be able to predict future
states, observations, and the task-relevant outcomes.

|  |  |  |
|----|----|----|
|  | 
``` math
\mathcal{G}=\Big(\underbrace{p_{\theta}(z_{t+1}\mid z_{t},a_{t})}_{\text{Dynamics}},\;\underbrace{p_{\theta}(o_{t}\mid z_{t})}_{\text{Observation}},\;\underbrace{p_{\theta}(r_{t}\mid z_{t},a_{t})}_{\text{Reward}},\;\underbrace{p_{\theta}(\gamma_{t}\mid z_{t},a_{t})}_{\text{Discount/Termination}}\Big)
``` |  |

This subsystem, which models state transitions, observations, rewards,
and terminations, serves as the foundation for the
<span class="ltx_text ltx_font_bold">Generation</span> property.

</div>

</div>

<div id="S2.SS2.SSS0.Px2" class="section ltx_paragraph">

#### The Interactive Loop ($`\mathcal{F},\mathcal{C}`$).

<div id="S2.SS2.SSS0.Px2.p1" class="ltx_para ltx_noindent">

To be more than a passive movie generator, the model must support a
closed interactive loop. For partially observable worlds, it requires an
*inference filter* ($`q_{\phi}`$) for the agent to interpret
observations in real-time, and a *policy* ($`\pi_{\eta}`$) for it to act
upon its understanding of the world, often paired with a value function
($`v_{\omega}`$) to evaluate trajectories.

|  |  |  |
|----|----|----|
|  | 
``` math
\mathcal{F}:\quad\underbrace{q_{\phi}(z_{t}\mid h_{t-1},o_{t})}_{\text{State Inference}},\qquad\mathcal{C}=\Big(\underbrace{\pi_{\eta}(a_{t}\mid z_{t},h_{t})}_{\text{Policy}},\;\underbrace{v_{\omega}(z_{t},h_{t})}_{\text{Value}}\Big)
``` |  |

This loop is what enables true
<span class="ltx_text ltx_font_bold">Interaction</span> and
<span class="ltx_text ltx_font_bold">Real-time Adaptation</span>.

</div>

</div>

<div id="S2.SS2.SSS0.Px3" class="section ltx_paragraph">

#### The Memory System ($`\mathcal{M}`$).

<div id="S2.SS2.SSS0.Px3.p1" class="ltx_para ltx_noindent">

Finally, to ensure coherence over time, the model needs a memory system
that allows past events to inform the future. This is formally captured
by a recurrent state, $`h_{t}`$, which is updated based on past memory,
the current inferred state, and the last action.

|  |  |  |
|----|----|----|
|  | 
``` math
\mathcal{M}:\quad\underbrace{h_{t}=f_{\psi}(h_{t-1},z_{t},a_{t-1})}_{\text{Memory Update}}
``` |  |

This component is the basis for the property of
<span class="ltx_text ltx_font_bold">Memory</span>.

</div>

<div id="S2.SS2.SSS0.Px3.p2" class="ltx_para ltx_noindent">

A detailed formalism of each component is provided in
Appendix <a href="https://arxiv.org/html/2510.20668v1#A1" class="ltx_ref"
title="Appendix A A Formalization of the Three Subsystems ‣ From Masks to Worlds: A Hitchhiker’s Guide to World Models"><span
class="ltx_text ltx_ref_tag">A</span></a>. This definition clarifies why
a system like a Unified Model (Stage II) is a precursor rather than a
true world model. While it may possess a powerful generative heart, it
typically lacks the dedicated interactive loop and explicit memory
system required to sustain a persistent, agent-inhabited world.

</div>

</div>

</div>

</div>

<div id="S3" class="section ltx_section">

## <span class="ltx_tag ltx_tag_section">3 </span>Stage I: Mask-based Models Across Modalities

<div id="S3.p1" class="ltx_para ltx_noindent">

The first stage in the evolution toward world models is the era of
*mask-based modeling*, where a system learns by reconstructing missing
or corrupted parts of its input. This paradigm, which can be summarized
as mask, infill, and generalize, has proven to be strikingly universal
across modalities. It provides a unified way of tokenizing,
representing, and pretraining large models, establishing the foundation
for all subsequent stages.

</div>

<div id="S3.SS1" class="section ltx_subsection">

### <span class="ltx_tag ltx_tag_subsection">3.1 </span>Language Modality

<div id="S3.SS1.p1" class="ltx_para ltx_noindent">

Masked language modeling (MLM) has played a foundational role in modern
natural language processing. BERT (Devlin et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib28"
class="ltx_ref">2019</a>) introduced bidirectional context prediction,
in which 15% of tokens in each input are randomly replaced with a
\[MASK\] token and predicted from the surrounding context. SpanBERT
(Joshi et al., <a href="https://arxiv.org/html/2510.20668v1#bib.bib52"
class="ltx_ref">2020</a>) refined this approach by masking contiguous
spans rather than isolated tokens, improving extraction and reasoning
tasks. Sequence-to-sequence variants such as MASS (Song et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib100"
class="ltx_ref">2019</a>), T5 (Raffel et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib92"
class="ltx_ref">2020</a>), and BART (Lewis et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib60"
class="ltx_ref">2019</a>) reformulated MLM as a denoising autoencoding
objective. ELECTRA (Clark et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib21"
class="ltx_ref">2020</a>) improved sample efficiency by replacing the
MLM objective with a discriminative replacement-detection task.

</div>

<div id="S3.SS1.p2" class="ltx_para ltx_noindent">

Beyond fixed-ratio masking, a line of non-autoregressive work introduces
dynamic masking and unmasking through iterative refinement. RoBERTa (Liu
et al., <a href="https://arxiv.org/html/2510.20668v1#bib.bib71"
class="ltx_ref">2019</a>) demonstrated that simply optimizing BERT’s
training recipe with more data and dynamic masking yielded significant
gains. Mask-Predict (Ghazvininejad et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib31"
class="ltx_ref">2019</a>) introduced iterative refinement, re-masking
low-confidence tokens over several passes. This concept culminated in
discrete diffusion models (Li et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib66"
class="ltx_ref">2022</a>; He et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib45"
class="ltx_ref">2022b</a>; Gong et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib32"
class="ltx_ref">2022</a>; Ou et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib81"
class="ltx_ref">2024</a>; Sahoo et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib96"
class="ltx_ref">2024</a>; Shi et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib98"
class="ltx_ref">2024</a>), which replace fixed masking with a
time-indexed noise schedule and train the model to iteratively denoise.
As demonstrated by industrial systems like Mercury (Inception Labs
et al., <a href="https://arxiv.org/html/2510.20668v1#bib.bib51"
class="ltx_ref">2025</a>) and Gemini Diffusion (DeepMind,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib26"
class="ltx_ref">2025</a>), this dynamic denoising paradigm has matured
to rival or exceed autoregressive baselines in both quality and
inference speed, solidifying the power of masking as a core generative
principle (Yu et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib124"
class="ltx_ref">2025c</a>; Li et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib65"
class="ltx_ref">2025c</a>).

</div>

</div>

<div id="S3.SS2" class="section ltx_subsection">

### <span class="ltx_tag ltx_tag_subsection">3.2 </span>Vision Modality

<div id="S3.SS2.p1" class="ltx_para ltx_noindent">

The masked image modeling (MIM) paradigm extended this principle to
perception. Early works established two main branches. For
representation learning, BEiT (Bao et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib8"
class="ltx_ref">2021</a>) and, especially, MAE (He et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib43"
class="ltx_ref">2022a</a>) created direct visual analogues of BERT,
reconstructing masked tokens or patches to learn powerful features. This
spurred a family of related works exploring different reconstruction
targets and self-distillation techniques (Xie et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib118"
class="ltx_ref">2022</a>; Zhou et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib128"
class="ltx_ref">2021</a>; Wei et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib111"
class="ltx_ref">2022</a>).

</div>

<div id="S3.SS2.p2" class="ltx_para ltx_noindent">

For generative modeling, MaskGIT (Chang et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib16"
class="ltx_ref">2022</a>) and MUSE (Chang et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib17"
class="ltx_ref">2023</a>) pioneered the use of masked infilling for
high-quality parallel image synthesis. This generative trajectory has
recently culminated in models like Meissonic (Bai et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib5"
class="ltx_ref">2024</a>), demonstrating that masked generative
transformers can achieve fidelity rivaling that of large diffusion
models while offering superior efficiency and control.

</div>

<div id="S3.SS2.p3" class="ltx_para ltx_noindent">

This mask-reconstruct-generalize principle scaled effectively to video.
VideoMAE (Tong et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib106"
class="ltx_ref">2022</a>) and MaskFeat(Wei et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib111"
class="ltx_ref">2022</a>) showed that high-ratio tube masking was a
data-efficient method for learning spatiotemporal representations,
confirming that masking could capture not just static scenes but also
their dynamics.

</div>

</div>

<div id="S3.SS3" class="section ltx_subsection">

### <span class="ltx_tag ltx_tag_subsection">3.3 </span>Other Modalities

<div id="S3.SS3.p1" class="ltx_para ltx_noindent">

The universality of the masking paradigm was confirmed by its rapid
adoption in other fields. In audio, models like wav2vec 2.0 (Baevski
et al., <a href="https://arxiv.org/html/2510.20668v1#bib.bib4"
class="ltx_ref">2020</a>), HuBERT (Hsu et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib47"
class="ltx_ref">2021</a>), WavLM (Chen et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib18"
class="ltx_ref">2022</a>), and Audio-MAE (Huang et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib48"
class="ltx_ref">2022</a>) applied masked prediction to latent speech
representations. In 3D domains, Point-BERT (Yu et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib125"
class="ltx_ref">2022</a>) and Point-MAE (Pang et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib83"
class="ltx_ref">2023</a>) adapted masking to point clouds. The principle
was even extended to structured data with models like GraphMAE (Hou
et al., <a href="https://arxiv.org/html/2510.20668v1#bib.bib46"
class="ltx_ref">2022</a>). These successes reinforced the view of
masking as a cross-domain general approach to self-supervised learning.

</div>

<div id="S3.SS3.p2" class="ltx_para ltx_noindent">

In summary, Stage I established the principle of masking as a universal
foundation for representation learning. While this unified the
pretraining paradigm, the models themselves remained specialized
architectures. The inability of these separate models to form a holistic
worldview motivated Stage II: the pursuit of a single, unified
architecture.

</div>

</div>

</div>

<div id="S4" class="section ltx_section">

## <span class="ltx_tag ltx_tag_section">4 </span>Stage II: Unified Models

<div id="S4.p1" class="ltx_para ltx_noindent">

Stage I established a universal paradigm for representation learning,
but the models themselves remained specialists locked within their own
modalities. Stage II takes the crucial next step: unifying the models
themselves. We define a unified model as a system that processes and
generates across different modalities with a shared backbone and the
same paradigm. By collapsing modality-specific pipelines, these models
simplify scaling, enable powerful cross-modal transfer, and represent
the first decisive synthesis on the path toward a true world model.

</div>

<div id="S4.SS1" class="section ltx_subsection">

### <span class="ltx_tag ltx_tag_subsection">4.1 </span>Representative Works

<div id="S4.SS1.p1" class="ltx_para ltx_noindent">

Leading unified modeling efforts span several trajectories,
distinguished by their foundational paradigm. We exclude simple glue
models that stitch different paradigms for different modalities, such as
using autoregression for text and diffusion for image, as well as models
limited to text generation without extending to image generation or
other modalities.

</div>

<div id="S4.SS1.p2" class="ltx_para ltx_noindent">

<span class="ltx_text ltx_font_bold">Extending Language Model
Pre-training: Language-Prior Modeling</span>. The dominant trajectory
has been to extend the paradigm of autoregressive large language models
(LLMs) (Radford et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib90"
class="ltx_ref">2019</a>; Brown et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib11"
class="ltx_ref">2020</a>; Sun et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib102"
class="ltx_ref">2024a</a>). This began by connecting pre-trained vision
encoders to frozen LLMs, as pioneered by BLIP-2 (Li et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib62"
class="ltx_ref">2023</a>) and popularized by LLaVA (Liu et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib69"
class="ltx_ref">2023b</a>;
<a href="https://arxiv.org/html/2510.20668v1#bib.bib70"
class="ltx_ref">2024</a>), which was built upon LLaMA (Touvron et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib107"
class="ltx_ref">2023</a>). This approach was further pushed into
grounded multimodal reasoning by Kosmos-2 (Peng et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib86"
class="ltx_ref">2023</a>) and into embodied reasoning by PaLM-E (Driess
et al., <a href="https://arxiv.org/html/2510.20668v1#bib.bib30"
class="ltx_ref">2023</a>). More recently, systems like the EMU family
(Sun et al., <a href="https://arxiv.org/html/2510.20668v1#bib.bib103"
class="ltx_ref">2024b</a>; Wang et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib110"
class="ltx_ref">2024</a>), Chameleon (Chameleon Team,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib15"
class="ltx_ref">2024</a>), VILA-U (Wu et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib114"
class="ltx_ref">2024</a>), and Janus-Pro (Chen et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib19"
class="ltx_ref">2025</a>) have advanced towards true end-to-end unified
generation, creating both text and images within a shared token space
and unified autoregressive paradigm. In parallel, a notable offshoot of
this trend is rooted in mask-based language modeling. LLaDA (Nie et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib79"
class="ltx_ref">2025</a>) abandons the autoregressive framework and
models text through a masked diffusion process with a single
Transformer. Its multimodal extension, MMaDA (Yang et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib120"
class="ltx_ref">2025</a>), introduces a unified discrete diffusion
architecture for text and image, a mixed chain-of-thought fine-tuning
strategy, and a policy-gradient RL algorithm (UniGRPO) to unify
reasoning and generation across modalities within a single model. More
recently, Lavida-O (Li et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib64"
class="ltx_ref">2025b</a>), OneFlow (Nguyen et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib78"
class="ltx_ref">2025</a>), and Lumina-DiMOO (Xin et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib119"
class="ltx_ref">2025</a>) have further improved overall performance and
introduced new capabilities.

</div>

<div id="S4.SS1.p3" class="ltx_para ltx_noindent">

<span class="ltx_text ltx_font_bold">Extending Vision Model
Pre-training: Visual-Prior Modeling</span>. A parallel effort, grounded
in vision-centric foundations, began along two paths. The first path
built upon latent diffusion models, the foundation laid by Stable
Diffusion (Rombach et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib95"
class="ltx_ref">2022</a>) was later generalized to a unified, joint
diffusion process over text and images in models like UniDiffuser (Bao
et al., <a href="https://arxiv.org/html/2510.20668v1#bib.bib7"
class="ltx_ref">2023</a>) and Rojas et al.
(<a href="https://arxiv.org/html/2510.20668v1#bib.bib94"
class="ltx_ref">2025</a>). The second path is built upon the masked
image modeling (MIM) paradigm, with models like Muddit (Shi et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib99"
class="ltx_ref">2025</a>) extending Meissonic (Bai et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib5"
class="ltx_ref">2024</a>) into a unified discrete diffusion system that
produces both images and captions within a shared architecture and
paradigm. Besides, UniDisc (Swerdlow et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib105"
class="ltx_ref">2025</a>) trained a unified discrete-diffusion model
from scratch for both language and vision modalities.

</div>

<div id="S4.SS1.p4" class="ltx_para ltx_noindent">

<span class="ltx_text ltx_font_bold">Industrial-Scale Unified
Systems</span>. At production scale, Gemini (Comanici et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib22"
class="ltx_ref">2025</a>) and GPT-4o (Hurst et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib50"
class="ltx_ref">2024</a>) unify language and vision modalities in a
single model, although not in a single paradigm. These demonstrate that
unified modeling has transcended research to become a foundational
industrial paradigm.

</div>

</div>

<div id="S4.SS2" class="section ltx_subsection">

### <span class="ltx_tag ltx_tag_subsection">4.2 </span>Benefits and Gaps

<div id="S4.SS2.p1" class="ltx_para ltx_noindent">

The primary benefit of Stage II is the reduction of fragmentation,
leading to powerful cross-modal transfer and emergent capabilities. This
paradigm now underpins productized multimodal interaction at scale, as
demonstrated by industrial systems like Gemini (Comanici et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib22"
class="ltx_ref">2025</a>) and GPT-4o (Hurst et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib50"
class="ltx_ref">2024</a>). However, despite the impressive progress of
language-prior unified models in interactive dialogue, visual-prior
unified models for text-to-image and text-to-video remain limited to
single-shot synthesis or stepwise editing. They lack the capacity for
continuous, real-time closed-loop interaction. Thus, while Stage II
unified architectures, the creation of truly dynamic and interactive
worlds remains an open challenge and motivates Stage III.

</div>

</div>

</div>

<div id="S5" class="section ltx_section">

## <span class="ltx_tag ltx_tag_section">5 </span>Stage III: Interactive Generative Models

<div id="S5.p1" class="ltx_para ltx_noindent">

Here, models are no longer static predictors or one-shot generators, but
participants in a closed action-perception loop, sustaining interaction
through low-latency response and action-conditioned evolution. We define
interactive generative models as systems whose outputs are conditioned
on streamed inputs or user actions, supported by internal state. We
explore this evolution across three distinct domains: language-based,
video-based, and scene-based. As mask-based interactive modeling remains
underexplored, we take an architecture-agnostic view and summarize
general interaction paradigms rather than mask-specific methods.

</div>

<div id="S5.SS1" class="section ltx_subsection">

### <span class="ltx_tag ltx_tag_subsection">5.1 </span>Language-based Worlds: Interaction as Narrative

<div id="S5.SS1.p1" class="ltx_para ltx_noindent">

Classic interactive fiction (IF) (Niesz & Holland,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib80"
class="ltx_ref">1984</a>; Montfort,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib76"
class="ltx_ref">2011</a>; Ammanabrolu et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib1"
class="ltx_ref">2020</a>) established the paradigm of text-driven worlds
where players interact through textual descriptions and actions. These
took several forms: parser-based games where the player types text
commands character by character, choice-based games where the player
selects from a set of predefined action options, and hypertext-based
games where the player clicks on links embedded in the narrative.
Choice-based visual novels, such as Memories Off (KID,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib56"
class="ltx_ref">2004</a>), exemplify emotionally branching narratives in
which player decisions directly affect relationships and endings. These
static worlds naturally evolved into benchmarks for artificial
intelligence. A significant line of research, supported by platforms
such as TextWorld (Côté et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib23"
class="ltx_ref">2018</a>) and Jericho (Hausknecht et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib41"
class="ltx_ref">2020</a>), has focused on training agents to master
them. In these settings, the world was a fixed puzzle to be solved, and
the locus of intelligence was the agent who navigated a static world,
not the world itself.

</div>

<div id="S5.SS1.p2" class="ltx_para ltx_noindent">

A fundamental shift occurred when large language models (LLMs) (Hurst
et al., <a href="https://arxiv.org/html/2510.20668v1#bib.bib50"
class="ltx_ref">2024</a>; Comanici et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib22"
class="ltx_ref">2025</a>) themselves became the world engine. AI Dungeon
(Latitude, <a href="https://arxiv.org/html/2510.20668v1#bib.bib59"
class="ltx_ref">2024</a>) pioneered this transition, dynamically
generating new narrative branches in response to free-form user prompts.
Players could explore unbounded story spaces limited only by imagination
and the model’s generative capacity. This marked the transition from
solving pre-authored worlds to co-creating open-ended ones, envisioning
a future in which visual novels such as Memories Off (KID,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib56"
class="ltx_ref">2004</a>) could be generated interactively, offering
unique storylines and relationships for each player.

</div>

</div>

<div id="S5.SS2" class="section ltx_subsection">

### <span class="ltx_tag ltx_tag_subsection">5.2 </span>Video-based and Scene-Based Worlds: Interaction as Experience

<div id="S5.SS2.p1" class="ltx_para ltx_noindent">

Interactive generation in video and spatial domains has progressed from
offline frame prediction to real-time, controllable simulation. Early
work on world models (Ha & Schmidhuber,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib39"
class="ltx_ref">2018</a>) used latent rollouts to “dream” trajectories
for policy training, demonstrating the potential of closed-loop
simulation. GameGAN (Kim et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib57"
class="ltx_ref">2020</a>) advanced this idea into a neural game engine
that renders successive frames from user input while implicitly learning
game rules from observation. User control evolved from stepwise action
selection in Playable Video Generation (PVG) (Menapace et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib73"
class="ltx_ref">2021</a>), through 3D scenes with camera and
multi-object control in Playable Environments (PE) (Menapace et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib74"
class="ltx_ref">2022</a>), to natural language prompts in Promptable
Game Models (PGM) (Menapace et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib75"
class="ltx_ref">2024</a>), which enabled the semantic-level direction of
play.

</div>

<div id="S5.SS2.p2" class="ltx_para ltx_noindent">

Building on these conceptual foundations, a decisive trajectory emerged
with the Genie series. Genie-1 (Bruce et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib12"
class="ltx_ref">2024</a>) learned latent action interfaces from
Internet-scale videos to create controllable 2D environments. Genie-2
(Parker-Holder et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib85"
class="ltx_ref">2024</a>) extended this capability to larger, quasi-3D
spaces, initialized from a single image and playable via standard
controls. Genie-3 (Ball et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib6"
class="ltx_ref">2025</a>) scaled further, producing real-time
text-to-world experiences at 720p and 24 fps with minutes of coherent
play, a marked shift from passive video generation to active
interaction.

</div>

<div id="S5.SS2.p3" class="ltx_para ltx_noindent">

Community and industrial efforts soon followed. Systems such as Oasis
(Decart et al., <a href="https://arxiv.org/html/2510.20668v1#bib.bib25"
class="ltx_ref">2024</a>), GameNGen (Valevski et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib108"
class="ltx_ref">2024</a>), Mineworld (Guo et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib36"
class="ltx_ref">2025</a>), and Matrix-Game (He et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib44"
class="ltx_ref">2025</a>) demonstrated
<span class="ltx_text ltx_font_italic">real-time</span> open
environments with emergent physics and streaming diffusion. For a
comprehensive overview, see the survey by Yu et al.
(<a href="https://arxiv.org/html/2510.20668v1#bib.bib123"
class="ltx_ref">2025b</a>).

</div>

<div id="S5.SS2.p4" class="ltx_para ltx_noindent">

Beyond frame synthesis, scene-based approaches emerged. World Labs
(World Labs, <a href="https://arxiv.org/html/2510.20668v1#bib.bib112"
class="ltx_ref">2024</a>) proposed large world models that generate
explorable 3D environments from a single image, enabling interactive
navigation through generated geometry and depth rather than sequential
video.

</div>

<div id="S5.SS2.p5" class="ltx_para ltx_noindent">

Taken together, these advances trace a trajectory from offline video
generators to real-time, action-conditioned world simulators. They
ultimately transform generative models into engines of interactive human
experiences.

</div>

</div>

<div id="S5.SS3" class="section ltx_subsection">

### <span class="ltx_tag ltx_tag_subsection">5.3 </span>Challenges

<div id="S5.SS3.p1" class="ltx_para ltx_noindent">

Despite the leap to real-time interaction, sustaining long-horizon
consistency remains unsolved. Two paradigms illustrate the tension:
explicit scene generators like NeRFs and Gaussian Splatting
(<span class="ltx_text ltx_font_italic">e.g.</span>, World Labs) offer
stable 3D navigation environments but depend on explicit spatial
modeling; implicit frame-by-frame generators offer flexibility but are
brittle, prone to losing context and hallucinating objects, especially
over extended play. The Genie series highlights this tradeoff: from
Genie-1’s short 16-frame memory (Bruce et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib12"
class="ltx_ref">2024</a>), to Genie-2’s object permanence (Parker-Holder
et al., <a href="https://arxiv.org/html/2510.20668v1#bib.bib85"
class="ltx_ref">2024</a>), to Genie-3’s few minutes of coherence (Ball
et al., <a href="https://arxiv.org/html/2510.20668v1#bib.bib6"
class="ltx_ref">2025</a>), progress is clear yet far from persistence.
At the object level, implicit video models rely on KV caches or control
signals to maintain identity, while explicit 3D approaches embed spatial
location directly but still struggle with dynamic elements, as explored
in 4D Gaussian Splatting. These challenges reveal a deeper gap: the
reactive action–perception loop enables interaction, but without
dedicated memory and state management, it cannot sustain persistent
worlds — the central theme of Stage IV.

</div>

</div>

</div>

<div id="S6" class="section ltx_section">

## <span class="ltx_tag ltx_tag_section">6 </span>Stage IV: Memory and Consistency

<div id="S6.p1" class="ltx_para ltx_noindent">

A world model that acts without memory is reactive yet forgetful. This
stage aims to endow models with mechanisms that sustain coherent states
across long horizons. The central question emerges: can world models not
only generate but also sustain coherent histories, preserve identities,
and resist drift? We organize this section around three questions: where
to anchor memory, how to extend its span and capacity, and how to govern
it to preserve consistency. Because mask-based persistent memory remains
underexplored and varies widely, we take an architecture-agnostic view,
focusing on memory and consistency rather than specific mechanisms.

</div>

<div id="S6.SS1" class="section ltx_subsection">

### <span class="ltx_tag ltx_tag_subsection">6.1 </span>Externalized Memory

<div id="S6.SS1.p1" class="ltx_para ltx_noindent">

Retrieval augments parametric models with non-parametric, often
editable, knowledge stores. Early explorations such as Neural Turing
Machines (Graves et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib33"
class="ltx_ref">2014</a>), Differentiable Neural Computers (Graves
et al., <a href="https://arxiv.org/html/2510.20668v1#bib.bib34"
class="ltx_ref">2016</a>), and End-to-End Memory Networks (MemN2N)
(Sukhbaatar et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib101"
class="ltx_ref">2015</a>) first explored learnable read–write memory
slots. While conceptually groundbreaking, their complexity gave way to
more pragmatic, decoupled designs. kNN-LM (Khandelwal et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib55"
class="ltx_ref">2019</a>), REALM (Guu et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib38"
class="ltx_ref">2020</a>), and RAG (Lewis et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib61"
class="ltx_ref">2020</a>) showed that conditioning on retrieved passages
could dramatically expand effective context while keeping knowledge
traceable and updatable. DPR (Karpukhin et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib53"
class="ltx_ref">2020</a>) and RETRO (Borgeaud et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib9"
class="ltx_ref">2022</a>) scaled this approach to dense retrievers and
trillion-token databases, rivaling far larger dense LMs while providing
traceable and updatable evidence.

</div>

<div id="S6.SS1.p2" class="ltx_para ltx_noindent">

Beyond simple retrieval, research has sought to make memory more
scalable and dynamic. Product Key Memory (PKM) (Lample et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib58"
class="ltx_ref">2019</a>) supported massive lookup capacity through
factorized keys; MemGPT (Packer et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib82"
class="ltx_ref">2023</a>) reframed LLMs as operating systems with
explicit virtual memory management; LONGMEM (Wang et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib109"
class="ltx_ref">2023</a>) extends KV caches beyond 65k tokens through
decoupled readers; and From RAG to Memory (Gutiérrez et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib37"
class="ltx_ref">2025</a>) extended retrieval into continual learning,
enabling dynamic knowledge updates without retraining. These systems
collectively signal a shift from retrieval as a tool to memory as a
co-evolving substrate.

</div>

</div>

<div id="S6.SS2" class="section ltx_subsection">

### <span class="ltx_tag ltx_tag_subsection">6.2 </span>Extending Capacity and Span

<div id="S6.SS2.p1" class="ltx_para ltx_noindent">

Parallel efforts seek to build persistence directly into the
architecture, moving beyond fixed-length attention windows. Within
Transformers, Universal Transformer (Dehghani et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib27"
class="ltx_ref">2018</a>) introduced depth-wise recurrence;
Transformer-XL (Dai et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib24"
class="ltx_ref">2019</a>) propagated segment states across windows; the
Compressive Transformer (Rae et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib91"
class="ltx_ref">2019</a>) down-sampled older activations. Subsequent
designs such as the Memorizing Transformer (Wu et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib115"
class="ltx_ref">2022</a>) and Recurrent Memory Transformer (RMT)
(Bulatov et al., <a href="https://arxiv.org/html/2510.20668v1#bib.bib13"
class="ltx_ref">2022</a>) attached associative key–value stores or
persistent memory tokens, reaching million-token horizons in practice;
Infini-attention (Munkhdalai et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib77"
class="ltx_ref">2024</a>) added a compressive long-term path for
unbounded streaming. In parallel, Perceiver-AR (Hawthorne et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib42"
class="ltx_ref">2022</a>) introduced a latent cross-attention
bottleneck, compressing long inputs into a compact representation and
enabling autoregression over 100k tokens across text, images, and music.
Together, this line of work represents a reformist trajectory that
extends attention through recurrence and compression.

</div>

<div id="S6.SS2.p2" class="ltx_para ltx_noindent">

A more radical line argues that persistence requires abandoning
quadratic attention entirely. Structured state-space and linear-time
models such as S4 (Lu et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib72"
class="ltx_ref">2023</a>), Mamba (Gu & Dao,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib35"
class="ltx_ref">2023</a>), and RetNet (Sun et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib104"
class="ltx_ref">2023</a>) replace attention with recurrent state updates
that achieve linear complexity and thereby, in principle, support
infinite context. Precursors such as Linear Transformers (Katharopoulos
et al., <a href="https://arxiv.org/html/2510.20668v1#bib.bib54"
class="ltx_ref">2020</a>), along with more recent variants such as Hyena
(Poli et al., <a href="https://arxiv.org/html/2510.20668v1#bib.bib88"
class="ltx_ref">2023</a>), have pointed in this direction by using
kernels and long-range convolutions. Together, this line of work
represents a revolutionary trajectory that abandons attention in favor
of continuous dynamical systems.

</div>

<div id="S6.SS2.p3" class="ltx_para ltx_noindent">

Scaling strategies and engineering refinements further extend these
capacities. LongNet (Ding et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib29"
class="ltx_ref">2023</a>) employs dilated attention for billion-token
contexts; Ring Attention (Liu et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib68"
class="ltx_ref">2023a</a>) distributes computation across devices for
million-token horizons; LSSVWM (Po et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib87"
class="ltx_ref">2025</a>) adapts state-space updates for long causal
video generation. Practical techniques such as ALiBi (Press et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib89"
class="ltx_ref">2021</a>), LongLoRA (Chen et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib20"
class="ltx_ref">2023</a>), and StreamingLLM (Zeng et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib126"
class="ltx_ref">2024</a>) retrofit long-context ability into existing
models. Together, this line of work represents a pragmatic trajectory
that extends persistence through scaling strategies and engineering
refinements.

</div>

<div id="S6.SS2.p4" class="ltx_para ltx_noindent">

Ultimately, these three trajectories — reformist, revolutionary, and
pragmatic — converge on the same goal: achieving genuine continuity,
creating models that can read a book, watch a film, or play for hours
without losing the thread.

</div>

</div>

<div id="S6.SS3" class="section ltx_subsection">

### <span class="ltx_tag ltx_tag_subsection">6.3 </span>Regulating Memory for Consistency

<div id="S6.SS3.p1" class="ltx_para ltx_noindent">

Persistence without discipline degenerates into drift. The nature of
this challenge depends critically on the underlying world
representation, which has largely followed two paradigms: implicit 2D
video frames and explicit 3D scenes.

</div>

<div id="S6.SS3.p2" class="ltx_para ltx_noindent">

In implicit, autoregressive video models, the primary challenge is
preventing two entangled failures: forgetting, where early content
fades, and drifting, where errors compound. Efforts to mitigate one
often aggravate the other (Zhang & Agrawala,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib127"
class="ltx_ref">2025</a>). The Genie series highlights this progression:
Genie-1 (Bruce et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib12"
class="ltx_ref">2024</a>) suffers from short memory and drifts after
only a few frames; Genie-2 (Parker-Holder et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib85"
class="ltx_ref">2024</a>) introduces object permanence and sustains
coherence for about a minute; Genie-3 (Ball et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib6"
class="ltx_ref">2025</a>) reaches emergent multi-minute consistency.
This underscores a broader challenge: autoregressively generating an
environment is fundamentally harder than producing a pre-rendered video,
since small inaccuracies accumulate over time. To tackle this, FramePack
(Zhang & Agrawala,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib127"
class="ltx_ref">2025</a>) uses keyframe anchoring and context
compression; Self-Forcing (Huang et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib49"
class="ltx_ref">2025</a>) and CausVid (Yin et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib121"
class="ltx_ref">2025</a>) impose stronger causal constraints;
Context-as-Memory (Yu et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib122"
class="ltx_ref">2025a</a>) retrieves overlapping past frames to
stabilize long video rollouts, and Mixture of Contexts (MoC) (Cai
et al., <a href="https://arxiv.org/html/2510.20668v1#bib.bib14"
class="ltx_ref">2025</a>) learns sparse routing policies that focus
attention on salient history.

</div>

<div id="S6.SS3.p3" class="ltx_para ltx_noindent">

Conversely, explicit 3D representations built on generative assets from
models like Trellis (Xiang et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib116"
class="ltx_ref">2025</a>) or TripoSG (Li et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib67"
class="ltx_ref">2025d</a>) inherently provide strong spatial
consistency. Here, the challenge shifts to representing dynamic changes
and long-term object states. Methods like WorldMem (Xiao et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib117"
class="ltx_ref">2025</a>), geometry-grounded spatial memory (Wu et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib113"
class="ltx_ref">2025</a>) and surfel-indexed view memory (VMem) (Li
et al., <a href="https://arxiv.org/html/2510.20668v1#bib.bib63"
class="ltx_ref">2025a</a>) leverage this explicit 3D structure to
maintain a coherent world state over time, including dynamic
representations that capture evolving geometry and supporting
revisitations across long horizons. Beyond perceptual consistency,
maintaining logical and factual coherence in reasoning remains crucial,
addressed by techniques that learn to critique their own outputs (Asai
et al., <a href="https://arxiv.org/html/2510.20668v1#bib.bib3"
class="ltx_ref">2024</a>).

</div>

<div id="S6.SS3.p4" class="ltx_para ltx_noindent">

The overarching lesson is that longer context alone is insufficient.
Consistency emerges from explicit policies over memory: what to write,
what to retrieve, how to update, and when to forget.

</div>

</div>

<div id="S6.SS4" class="section ltx_subsection">

### <span class="ltx_tag ltx_tag_subsection">6.4 </span>Summary

<div id="S6.SS4.p1" class="ltx_para ltx_noindent">

Stage IV reframes generation as stateful computation. Externalized
memory makes knowledge editable. Architectural persistence makes it
durable. Consistency policies make it reliable. At production scale,
multimodal systems such as Gemini (Comanici et al.,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib22"
class="ltx_ref">2025</a>) and Claude (Anthropic,
<a href="https://arxiv.org/html/2510.20668v1#bib.bib2"
class="ltx_ref">2024</a>) extend these ideas, sustaining million-token
contexts across text, audio, and video and coupling long horizons with
reasoning for agentic workflows.

</div>

<div id="S6.SS4.p2" class="ltx_para ltx_noindent">

A deeper question remains. Are elaborate memory systems fundamental
solutions, or are they sophisticated workarounds for the current
constraints of hardware and data? The existence of models with massive,
brute-force context windows suggests that some memory problems might
simply dissolve with sufficient scale, much like how larger models
unlocked emergent abilities. Similarly, consistency failures may also
stem from limited data diversity or flaws in the data itself, such as
contradictory or erroneous text and videos that are only a few seconds
long. The answer will determine whether persistence in world models
emerges as a natural property of scale, or as the product of carefully
engineered memory discipline. When we ask whether world models can dream
consistently, the answer we seek is not just an engineering target but a
deeper understanding of the interplay among architecture, scale, and
data.

</div>

</div>

</div>

<div id="S7" class="section ltx_section">

## <span class="ltx_tag ltx_tag_section">7 </span>Stage V: Towards True World Models

<div id="S7.p1" class="ltx_para ltx_noindent">

The preceding stages constructed the necessary components: a universal
generative paradigm (I), a unified architecture (II), a real-time
interactive loop (III), and a persistent memory system (IV). Stage V is
not the addition of another component, but the synthesis of these parts
into a cohesive, autonomous whole. A true world model is not merely a
sophisticated simulator controlled by a user; it is a self-sustaining
computational ecosystem. Its defining properties are not just programmed
but emergent. We show that this synthesis gives rise to three defining
properties: Persistence, Agency, and Emergence.

</div>

<div id="S7.SS1" class="section ltx_subsection">

### <span class="ltx_tag ltx_tag_subsection">7.1 </span>The Threshold: Persistence, Agency, and Emergence

<div id="S7.SS1.p1" class="ltx_para ltx_noindent">

A true world model ceases to be a program one runs, but a world one
enters. Its defining properties are:

</div>

<div id="S7.SS1.p2" class="ltx_para ltx_noindent">

- <span id="S7.I1.i1"><span class="ltx_tag ltx_tag_item">•</span></span>
  <div id="S7.I1.i1.p1" class="ltx_para">

  <span class="ltx_text ltx_font_bold">Persistence</span>: The world’s
  state and history exist independently of any single user session,
  accumulating consequences over time. It has a past that can be
  revisited and a future that unfolds continuously. This is the ultimate
  fulfillment of the Memory System ($`\mathcal{M}`$), transforming the
  property of Memory into an enduring reality.

  </div>
- <span id="S7.I1.i2"><span class="ltx_tag ltx_tag_item">•</span></span>
  <div id="S7.I1.i2.p1" class="ltx_para">

  <span class="ltx_text ltx_font_bold">Agency</span>: The world is
  inhabited by multiple, goal-directed agents (human or AI) that
  interact within a shared context. This property is enabled by the
  Interactive Loop ($`\mathcal{F},\mathcal{C}`$), elevating the
  properties of Interaction and Adaptation into a multi-agent society.

  </div>
- <span id="S7.I1.i3"><span class="ltx_tag ltx_tag_item">•</span></span>
  <div id="S7.I1.i3.p1" class="ltx_para ltx_noindent">

  <span class="ltx_text ltx_font_bold">Emergence</span>: The world’s
  macro-level dynamics arise from the micro-level interactions of its
  agents and underlying rules, rather than being explicitly scripted.
  The model becomes a crucible for discovering unforeseen social
  structures, behaviors, and causal chains. This is the critical
  synthesis that occurs only when the Generative Heart
  ($`\mathcal{G}`$), Interactive Loop ($`\mathcal{F},\mathcal{C}`$), and
  Memory System ($`\mathcal{M}`$) operate in unison over time.

  </div>

</div>

</div>

<div id="S7.SS2" class="section ltx_subsection">

### <span class="ltx_tag ltx_tag_subsection">7.2 </span>The Frontier: Three Defining Challenges

<div id="S7.SS2.p1" class="ltx_para ltx_noindent">

The path to this threshold is defined by three fundamental, unsolved
research problems. These are not merely technical hurdles, but grand
challenges that constitute the frontier of the field.

</div>

<div id="S7.SS2.SSS0.Px1" class="section ltx_paragraph">

#### The Coherence Problem (Evaluation).

<div id="S7.SS2.SSS0.Px1.p1" class="ltx_para ltx_noindent">

For conventional models, fidelity is measured against external ground
truth. A true world model, however, writes its own history. The
challenge is to evaluate the “truth” of a self-generating reality: to
formalize and measure its internal logical, causal, and narrative
coherence, and to define what it means for such a world to be
consistent.

</div>

</div>

<div id="S7.SS2.SSS0.Px2" class="section ltx_paragraph">

#### The Compression Problem (Scaling).

<div id="S7.SS2.SSS0.Px2.p1" class="ltx_para ltx_noindent">

An ever-growing history risks computational collapse. The challenge is
to learn causally sufficient state abstractions that preserve
consequence while discarding noise, approaching the
information-theoretic bounds of predictive representation. Yet even with
abstraction, long-horizon dynamics may be computationally irreducible,
forcing us to treat world models not only as engineered systems but as
objects of scientific observation.

</div>

</div>

<div id="S7.SS2.SSS0.Px3" class="section ltx_paragraph">

#### The Alignment Problem (Safety).

<div id="S7.SS2.SSS0.Px3.p1" class="ltx_para ltx_noindent">

An autonomous, persistent world model is a technology with profound
societal implications. The alignment challenge for a true world model
operates on two distinct levels. At its base, the model itself can be
viewed as a single environment whose generating process must align with
human values. However, the complexity arises when this model becomes the
substrate for a multi-agent society. The alignment problem then becomes
squared: it requires aligning not only the world’s underlying laws (the
substrate), but also the emergent, unpredictable dynamics of the agents
interacting within it. This is the harder challenge, distinguishing a
true world model from a mere single-environment simulator.

</div>

</div>

</div>

<div id="S7.SS3" class="section ltx_subsection">

### <span class="ltx_tag ltx_tag_subsection">7.3 </span>The Horizon: From Simulator to Scientific Instrument

<div id="S7.SS3.p1" class="ltx_para ltx_noindent">

The journey detailed in this paper — from masks to worlds — has been
about forging a new kind of technology. Yet, the ultimate promise of a
true world model lies beyond its function as a simulator for
entertainment or training.

</div>

<div id="S7.SS3.p2" class="ltx_para ltx_noindent">

Once a world model crosses the threshold of persistence, agency, and
emergence, it transforms from a technological artifact into a new kind
of scientific instrument. It becomes a computational crucible for
running experiments on complex adaptive systems such as economies,
cultures, and cognitive ecosystems that are impossible to conduct in
reality.

</div>

<div id="S7.SS3.p3" class="ltx_para ltx_noindent">

The quest for true world models, therefore, is not merely an engineering
endeavor. It is a pursuit of the ultimate tool for understanding
complexity itself. The narrow road leads here: to a future where we
build worlds not to escape our own, but to comprehend it.

</div>

</div>

</div>

<div id="S8" class="section ltx_section">

## <span class="ltx_tag ltx_tag_section">8 </span>Conclusion: Building Living Worlds

<div id="S8.p1" class="ltx_para ltx_noindent">

This paper has charted a narrow road: a logical progression from the
universal paradigm of masking to the threshold of a new reality. We have
shown that this path is defined by the sequential mastery of three
fundamental capabilities: unified generation, real-time interaction, and
persistent memory. These are not ends in themselves, but the necessary
foundations for worlds that can truly be called living worlds that
persist with their own history, that are inhabited by goal-directed
agents, and that give rise to unforeseen emergence.

</div>

<div id="S8.p2" class="ltx_para ltx_noindent">

The pursuit of isolated benchmarks for static tasks is a detour. The
true frontier lies in embracing the architectural and theoretical
commitments required to build these self-sustaining computational
ecosystems. Therefore, the great choice ahead is whether we build worlds
as mere tools for entertainment and escapism, or as scientific
instruments for comprehending our complexity. The narrow road we have
charted leads to this horizon: a future where we forge not just better
models, but new mirrors in which to see ourselves.

</div>

## <span class="ltx_tag ltx_tag_appendix">Appendix A </span>A Formalization of the Three Subsystems

<div id="A1.p1" class="ltx_para ltx_noindent">

This appendix provides a detailed breakdown of the components formalized
in
Section <a href="https://arxiv.org/html/2510.20668v1#S2" class="ltx_ref"
title="2 What is a World Model? ‣ From Masks to Worlds: A Hitchhiker’s Guide to World Models"><span
class="ltx_text ltx_ref_tag">2</span></a>. We consider a standard
partially observable Markov decision process (POMDP) formulation where
at each timestep $`t`$, an agent takes an action $`a_{t}`$, receives an
observation $`o_{t}`$, and a reward $`r_{t}`$. The world terminates
based on $`\gamma_{t}`$. The model maintains a latent belief state
$`z_{t}`$ and a deterministic memory state $`h_{t}`$.

</div>

<div id="A1.SS0.SSS0.Px1" class="section ltx_paragraph">

#### The Generative Heart ($`\mathcal{G}`$).

<div id="A1.SS0.SSS0.Px1.p1" class="ltx_para ltx_noindent">

This subsystem models the world’s underlying generative process and
comprises three components:

- <span id="A1.I1.i1"><span class="ltx_tag ltx_tag_item">•</span></span>
  <div id="A1.I1.i1.p1" class="ltx_para">

  <span class="ltx_text ltx_font_bold">Dynamics Model</span>
  $`p_{\theta}(z_{t+1}\mid z_{t},a_{t})`$: Predicts the next latent
  state given the current state and an action. This is the core of the
  model’s ability to “dream” futures.

  </div>
- <span id="A1.I1.i2"><span class="ltx_tag ltx_tag_item">•</span></span>
  <div id="A1.I1.i2.p1" class="ltx_para">

  <span class="ltx_text ltx_font_bold">Observation Model</span>
  $`p_{\theta}(o_{t}\mid z_{t})`$: Maps a latent state back to a sensory
  observation (<span class="ltx_text ltx_font_italic">e.g.</span>, a
  video frame), grounding the latent space in perceptible reality.

  </div>
- <span id="A1.I1.i3"><span class="ltx_tag ltx_tag_item">•</span></span>
  <div id="A1.I1.i3.p1" class="ltx_para ltx_noindent">

  <span class="ltx_text ltx_font_bold">Outcome Model</span>
  $`p_{\theta}(r_{t},\gamma_{t}\mid z_{t},a_{t})`$: Predicts
  task-relevant outcomes like rewards and termination signals from the
  latent state.

  </div>

</div>

</div>

<div id="A1.SS0.SSS0.Px2" class="section ltx_paragraph">

#### The Interactive Loop ($`\mathcal{F},\mathcal{C}`$).

<div id="A1.SS0.SSS0.Px2.p1" class="ltx_para ltx_noindent">

This subsystem enables a closed-loop exchange between an agent and the
world model. It consists of:

- <span id="A1.I2.i1"><span class="ltx_tag ltx_tag_item">•</span></span>
  <div id="A1.I2.i1.p1" class="ltx_para">

  <span class="ltx_text ltx_font_bold">Inference Model (Filter)</span>
  $`q_{\phi}(z_{t}\mid h_{t-1},o_{t})`$: Infers the current latent
  belief state $`z_{t}`$ from the new observation $`o_{t}`$ and past
  memory $`h_{t-1}`$.

  </div>
- <span id="A1.I2.i2"><span class="ltx_tag ltx_tag_item">•</span></span>
  <div id="A1.I2.i2.p1" class="ltx_para ltx_noindent">

  <span class="ltx_text ltx_font_bold">Control Model (Policy &
  Value)</span>
  $`\pi_{\eta}(a_{t}\mid z_{t},h_{t}),v_{\omega}(z_{t},h_{t})`$: The
  policy selects the next action based on the current belief and memory,
  while the value function estimates future outcomes, guiding the
  policy.

  </div>

</div>

</div>

<div id="A1.SS0.SSS0.Px3" class="section ltx_paragraph">

#### The Memory System ($`\mathcal{M}`$).

<div id="A1.SS0.SSS0.Px3.p1" class="ltx_para ltx_noindent">

This subsystem ensures long-horizon coherence. It has one core
component:

- <span id="A1.I3.i1"><span class="ltx_tag ltx_tag_item">•</span></span>
  <div id="A1.I3.i1.p1" class="ltx_para ltx_noindent">

  <span class="ltx_text ltx_font_bold">Memory Update Model</span>
  $`h_{t}=f_{\psi}(h_{t-1},z_{t},a_{t-1})`$: Updates the deterministic
  memory state based on the previous memory, the inferred state, and the
  last action, creating a persistent representation of history.

  </div>

</div>

<div id="A1.SS0.SSS0.Px3.p2" class="ltx_para ltx_noindent">

This component-based formalization provides a unified lens through which
to view the historical evolution of the field, from early
control-oriented models that focused on specific components
(<span class="ltx_text ltx_font_italic">e.g.</span>, Ha & Schmidhuber
(<a href="https://arxiv.org/html/2510.20668v1#bib.bib39"
class="ltx_ref">2018</a>)) to modern generative systems that aim to
integrate them all. It forms the analytical foundation for the
five-stage roadmap presented in this paper.

</div>
