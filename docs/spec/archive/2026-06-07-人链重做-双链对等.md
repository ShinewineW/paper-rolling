# 人链重做:让"给人看"成为与 ARA 对等的第一类链

> **日期**: 2026-06-07
> **状态**: 草稿
> **作者**: Claude(Opus 4.8)+ ShinewineW
> **基准版本**: `paper-rolling@主分支`(未提交工作树)
> **目的**: 修复"给人看的报告又薄又假大空"——把人链从 ARA 的薄投影,重做成与 ARA 对等的、为可读性独立优化的第一类生成链。
> 范围: `.claude/skills/paper-landscape/`(seams 契约 + branch1 + analyzer 产出 + 审计门)、`attn_sink/cosmos_trial/seams.py`(运行时 seam 实现)、tests、docs/adr

## 一、问题(已验证)

- person_vault 报告实测 **~805 中文字**,设计目标 **3000–6000 字**;参考仓人看报告样例 **~30KB**。差一个数量级。
- branch1 只渲染 ARA 的 3/18 字段(claims/architecture/algorithm),且多节是**占位/套话**:写死的 `Input→Model→Output` mermaid、模板"逐步推导"、通用"玩具例子"。
- 根因:branch1 是 paper-rolling **原创**且**故意机械化**的——为稳过锚点硬门,把丰富段落都做成"无数字示意"。它是 ARA 的安全薄投影,不是对等的链。

## 二、目标架构:一源两链

事实源(冻结的 `{ID}.md`)→ 两条**对等**链:

| 链 | 消费者 | 形态 | 产物 | 状态 |
|----|--------|------|------|------|
| ARA 链 | AI | 结构化、信息致密 | `ai_package/` | 已有,基本合格(小幅增强) |
| **人链** | 人 | **形象生动、语言流利、长文** | `person_vault/` | **重做** |

关键原则:人链**不再是 ARA 的渲染**,而是一个**独立的写作 seam**,以"已通过 G2 数据保真门的 ARA bundle"为事实底座、以 `{ID}.md` 为引文来源,**专为可读性生成**。

## 三、设计要点

### A. 新增写作 seam(第二条链的发动机)
- 新 seam `write_report(analysis_bundle, md_text, candidate) -> 中文报告 markdown`。
- 独立 LLM 调用(Sonnet / effort=high,质量优先),提示词**只关心"讲清楚、讲生动、讲流利"**,与 analyzer 的"结构化抽取"职责分离(对应你的"两条链各自优化")。
- 事实底座 = ARA bundle(已过 G2,数字可信);引文 = `{ID}.md`。**禁止编造**(沿用 analyzer 的忠实规则:数字逐字、专名/公式原样、叙述中文)。

### B. 接地不破坏(关键约束)
- 锚点硬门(吸收-D1)仍在:任何"数字+指标线索"句必须带 `<!--ref--><!--anchor:quote:-->`。
- 方案:**机械后置锚定**(复用现有 `_find_in_md`)——写作 seam 产出忠实富文 → branch1 对每个出现在 MD 的数字自动注入锚点 → lint 校验。比"让 LLM 自己写锚点格式"更稳。
- ARA 数字已过 G2,绝大多数能在 MD 找到可锚;tolerant 容忍的可疑数字(AUDIT_FLAGS)在人报告里**降级为定性表述**或省略,不进硬性能宣称。

### C. 干掉"假大空"(b 部分,触及 analyzer 产出契约)
- analyzer bundle 增结构化字段:
  - `derivation_steps: [{step, from, to, why}]` —— 真推导步骤(替换模板套话);
  - `architecture_graph: {nodes:[{id,label,kind}], edges:[{from,to,label}]}` —— 真架构图(替换写死的 Input→Model→Output);
  - `math_toy_example` —— 真玩具例子(已有字段但常缺,强制产出)。
- branch1/写作 seam 渲染这些真实结构;**删除**写死的 mermaid 骨架与模板推导。

### D. 富报告章节(目标 3000–6000+ 字,中文流畅)
摘要导读 / 问题背景与动机 / 核心概念逐条 / 方法与架构(真图+叙述)/ 算法与推导(真步骤+比喻+玩具例子)/ 实验与对比(表+解读)/ 相关工作 / 探索历程(exploration_tree 叙事化)/ 局限与边界 / 趋势与定位。
