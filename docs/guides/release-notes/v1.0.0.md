## 一、这是什么

paper-rolling 是一套**论文知识引擎,同时也是它产出的知识库本身**。引擎是一条纯函数、确定性、依赖注入的 Python 管道:发现领域内权威 + 最新论文 → 把 PDF 转成 Markdown → 为每篇论文产出两件配对产物,经两道对抗式审计门把关后原子发布,并把知识沉淀进同一个仓库。

两件产物每篇成对、1:1 配对:

- **人链报告**(`person_vault/<key>/report.md` + `report.html`)——给人读的中文图文深度解读。
- **AI 知识包 / ARA**(`ai_package/<key>/ara/`)——给 AI 消费的结构化已验证知识包(论断、概念、实验、证据表、码链指针、封印报告)。

v1.0.0 是**第一个稳定版**:语料 33 篇全部通过机器门 + 最强模型终审,引擎通过完整测试与跨模型验收。

## 四、引擎架构

**核心心智模型——纯核 + 运行时装配**:引擎是一个纯核,由运行时(一个 Claude Code agent)在调用时注入"接缝(seams)"。**没有运行 campaign 的 `__main__`**——直接执行 `run_campaign.py` 只打印用法。这保证了引擎可被完整测试(用 fake seams),且对抗式保证无法被绕过。

管道顺序(每篇串行的"轮辐 spoke"):

```
发现 → 摄入 → branch2(ARA)→ G2 → branch1(人链报告)→ G3 封印 → 原子发布
```

- **发现**(`scripts/discovery/`):多源(OpenAlex / S2 / arXiv / DBLP / HF Papers)→ 去重 → 多信号 OR 权威排序(ADR-0001,不设单一引用硬门)。
- **摄入**(`scripts/ingest/`):2 层 PDF→MD。Tier-1 = arXiv HTML + `pandoc`(快);Tier-2 = MinerU CLI(CPU)。MD 唯一真值 + 来源契约 + 方程块哈希。
- **双产物**:branch2(ARA)与 branch1(人链报告)**原子发布(both-or-neither)**,同一 vault_key 配对。
- **两道对抗门**:**G2** = 数字保真(多票怀疑者硬阻断编造数字,真值隔离);**G3** = 封印(锚点 + 方程保真 + 蕴含 + 6 维严谨性)。
- **状态**:`_ledger/processed_ledger.yaml` 单写者、追加式、LS-1 锁。失败篇移入 `_failed/` 现场(ADR-0011 永不反射删 token-expensive 的 ARA),可经 `revival.py` 批次复活赛。

**LLM 接缝路由**:`config/llm.yaml` 把每个 seam 路由到提供商并钉死调用参数(tier / effort / timeout)。每个 seam 经 `StrictProvider` 包裹——提供商失败即 `EngineAbort` 大声中止、**无回退**(成本守卫:不静默降级到 Claude Code 订阅)。本版 analyzer 路由到接地的本地 agent(codex `codex exec`),G2/G3/writer 路由到 OpenAI 兼容 API,faithfulness 与可选 web_search 路由到 claude-code。

## 五、本版关键新增与变更

v1.0.0 在既有引擎之上落定了以下能力(本轮):

1. **ADR-0013 终审修订层全量执行完成**——可选、操作者触发、默认关、不接进 `/loop` 的"机器门后收尾验收层"。一轮跑完后,主会话逐篇派 Opus 子 agent 对已发布产物对照源 MD「修订(REVISE)/确认(CLEAN)/回炉(FAIL)」。本版已对**全部 33 篇**执行(标记落在 `ai_package/*/ara/final_review.json`),且引擎侧只有纯确定性函数、不新增 LLM seam(成本守卫不变)。
2. **`resolve_paper_paths` 单一路径解析器**(`scripts/output/naming.py`)——一篇论文的产物散在 corpus(键 `{idbase}_{Slug}`)与 person_vault/ai_package(键 `{date}_{Name}_{idbase}`)两套命名下,均不按裸 idbase。新增的解析器按盘扫描收拢"idbase → 全部产物路径"的唯一权威,外部工具不再各自手工 glob。
3. **Tier-2 `content_list.json` durability 修复**(`scripts/ingest/ingest.py`)——本地 Tier-2(MinerU)曾只把 `content_list.json` 留在 gitignored 的 `mineru/` 子树,提交后丢失、G3 方程门无法复跑。现 `_finalize` 拷贝到可跟踪的 `corpus/<id>/content_list.json`。
4. **preflight `final-review:runtime` 自检通告**(`scripts/preflight.py`)——引擎从不自主运行;终审门无引擎侧评审员。非 Claude-Code 运行时(如纯 API)以 `[WARN]`(不阻断)如实通告"终审门无自主评审员",由操作者决定路由到自有 provider 或显式跳过。
5. **终审提醒 Stop hook + 冷却**(`.claude/hooks/final_review_reminder.py` + `.claude/settings.json`)——项目级 Stop hook,在主会话交还控制权时,若有"过机器门但未终审"的论文则提醒;约每 30 分钟每会话最多一次(冷却),内部错误一律静默放行、绝不阻断退出。
6. **状态卡显示终审聚合**——`scripts.status --card` 的进度漏斗新增 `终审 N`。
7. **DriveWAM(2605.28544)全流水线验证**——经正常 campaign 触发,从发现/摄入一路跑到 G3 发布 + 终审,作为稳定版的端到端实证(`done=1, failed=0`)。

## 七、前置、上手与使用

**前置依赖**(`scripts.preflight` 会逐项检查并给出修复命令,跑任何东西前先跑它):

- `uv`(Astral Python 工具链)——环境与依赖管理;装法见 Astral 官方。
- `pandoc`——Tier-1(arXiv HTML→MD);`brew install pandoc` 或 GitHub release 二进制。
- `mineru`——Tier-2(PDF→MD,CPU,对内存有要求);安装与"装了≠能跑"的排查见 [`docs/guides/mineru-setup.md`](../mineru-setup.md)。

**安装 + 自检 + 看状态**(标准模块 CLI 需带 `PYTHONPATH` 前缀):

```bash
uv sync --group dev                  # 建项目本地 .venv
uv run pytest                        # 全套测试 = 可执行规范
PYTHONPATH=.claude/skills/paper-landscape uv run python -m scripts.preflight        # 环境自检(先跑;含 LLM provider 探活 + ingest 冒烟)
PYTHONPATH=.claude/skills/paper-landscape uv run python -m scripts.status --card    # 语料合规态(权威)
PYTHONPATH=.claude/skills/paper-landscape uv run python -m scripts.output.check_ara_bundle   # ARA 结构回归门
```

**跑第一次 campaign**:引擎不可裸 shell 调用(设计如此)——在仓库根 `~/Coding/paper-rolling/` 内,由 Claude Code 的 `/paper-landscape` 技能或 `/loop` tick 驱动:它经 Hard Gate(`config/campaign.yaml`:确认 topic + 每 tick N)确立后,调 `run_campaign(...)` 注入 seams 运行。完整运行契约(seam 接线、`/loop` 节奏、管道顺序)见 [`.claude/skills/paper-landscape/SKILL.md`](../../../.claude/skills/paper-landscape/SKILL.md)。终审收尾(可选)见 `sub-skills/final-review/SKILL.md`。

**LLM 配置**:`config/llm.yaml` 把每个 seam 路由到 provider 并钉死调用参数;密钥在 gitignored `.env`。本版 analyzer→本地 `codex`(`codex exec`),G2/G3/writer→**操作者自备的 OpenAI 兼容 API 端点**(本版示例 `HELLOROBOTAXI_API_KEY` 即操作者自有 API,`OPENCODE_API_KEY` 同理),faithfulness + 可选 web_search→`claude-code`。换 provider 只改 `llm.yaml`、不动代码;字段含义与四种 provider 类型见仓库根 `README.md` 与 `.claude/CLAUDE.md` 的"LLM provider routing"节。

**权威文档**:仓库根 `README.md`(模块全图 / env / gotcha)、`.claude/skills/paper-landscape/SKILL.md`(运行契约)、[`docs/INDEX.md`](../../INDEX.md)(文档地图 + 治理)。

## 八、已知限制与非缺陷(正常,非缺口)

- **Tier-1 方程保真是"信任而非验证"**:仅 Tier-2(MinerU)有真正的机械方程检查;pandoc 路径按构造合成一份通过的 `content_list.json`。
- **部分论文无公开代码仓**:`code_ref` 如实写"未找到公开仓库",非每篇都开源代码。
- **长篇人类报告的残余小瑕疵被容忍且标注**:每篇报告开篇的 `## 评价` 段显式披露该篇已知小瑕疵——这是设计内的诚实披露。终审是最强模型判定,非数学证明:不宣称"零文字瑕疵"。
- **引擎不自主运行**:必然由 AI 运行时驱动并注入 seams;"不能裸 shell 跑"是设计,不是缺陷。

## 九、延后项 / 后续路线

- **命名键统一**:corpus 键(`{idbase}_{Slug}`)与 vault 键(`{date}_{Name}_{idbase}`)是两套刻意分离的命名空间;v1.0.0 以 `resolve_paper_paths` 收拢解析作为过渡。是否统一为单一可读键已记录、留作后续 spec(届时解析器可退化)。
- **账本绝对路径**:`_ledger` 记录绝对产物路径(完整性靠存储字段校验、0 divergence),可移植性弱于相对路径;与命名统一同属一次后续迁移。
- 能力 / 可信度路线:见 `docs/guides/ROADMAP.md`。

## 十、约束与约定(运维须知)

- **无硬编码密钥**:所有 token(`OPENCODE_API_KEY` / `HELLOROBOTAXI_API_KEY` / `HF_TOKEN`)从 gitignored `.env` 读取,源码不携带。
- **无静默回退到 Claude Code 订阅**:必需 LLM seam 无默认 provider、无 fallback,失败即大声中止(成本守卫)。
- **单写者账本**:hub(`/loop` tick)与复活赛驱动均持 LS-1 锁、互斥,绝不并发写账本。
- **追踪 = 产物,忽略 = 可再生输入**:converted `corpus/{ID}/{ID}.md` + `.md_contract.json` + `content_list.json`、`person_vault/`、`ai_package/`、`_ledger/`、`landscapes/`、引擎 + `config/` 入库;原始 `*.pdf`、MinerU `images/`、`.env`、`.venv` 忽略。
