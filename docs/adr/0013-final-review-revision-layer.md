# 0013 — 终审修订层:可选的、主会话(workflow)对已发布产物的批次级「修订或回炉」

> **状态**: accepted — 2026-06-14(设计定稿,经 `/mp-grill-me` 七题拷问;实现计划见
> `docs/spec/2026-06-14-final-review-revision-layer-impl.md`)。
> **关联**: 建立在 ADR-0012(人链不设硬门、ARA 为唯一真值)、ADR-0007(外科 re-emit、复用上游)、
> ADR-0011(ARA 永不反射删、失败现场单向汇)之上;**不修改**这三者。

## 背景

引擎的所有 LLM seam 都路由到**便宜/快**的外部后端(分析器→codex、双门+写手→内网 qwen、忠实判官
→haiku),整条流水线与主会话**解耦**(可 headless/cron)——这是成本守卫 + 无人值守自动化的基石。
代价是:机器门(数字门 / 最终门)过了 **不等于** "最强模型逐篇确认过了"。实证:本轮 Phase C 用强
模型(Opus 主会话 + 子 agent)对照原文复审 5 篇,机器门**全过**的产物里仍抓出 2 个真缺陷(写手思维
链泄漏、码链 prose 与 SSOT 对不上)。需要一个把"门过了"提升到"最强模型确认/修订过了"的收尾层。

## 决策

**新增一个可选的「终审修订」层:操作者在一轮 `/loop` 跑完后按需触发,主会话用 Workflow 派每篇一个
继承主会话 Opus、各自独立上下文的子 agent,对已发布产物对照源 MD 做「修订或回炉」。** 七项定稿:

1. **全文件可改**:子 agent 可改人链报告与 ARA 全部文件(信任最强模型 + 有论文做基底)。
2. **目的是改不是判**:小瑕疵子 agent **直接外科修**(干净聚焦的子 agent 上下文 ≥ 被对话稀释的主
   会话);**不设 LLM 验收**,只在 REVISE 后跑一道**纯机械回归**(`check_ara_bundle` + status 合规)
   防"改崩"。
3. **修订 vs 回炉 = 「基底 vs 瓦砾」判据**:有可信基底就打补丁;**读错论文 / 整体胡说 / 重写级**这三类
   = 无基底 → FAIL。存疑偏 REVISE。FAIL 暂封闭于这三类。
4. **改了封印内容 → 信任 + 盖来源章,不重跑门**:`final_review.json` sidecar 记 `{verdict, provenance}`,
   让 `passes_seal2` 不沦为对旧内容的沉默断言;sidecar 同时是幂等键(重跑跳过已审)。
5. **FAIL 走复活赛**:把已发布产物**降级到失败现场**、现场强制 **branch2 根** → 复活赛**重新分析**
   (不复用坏 ARA),与 ADR-0011/0007 的恢复线统一。
6. **agent 改内容、主会话碰账本**:子 agent 各自上下文下笔、并行(每篇只碰自己的文件);账本 / 失败
   现场 / 复活赛只由主会话**串行**写(单写者不变量 LS-1)。
7. **是验收层不是门**:非确定、会话级、操作者触发、**不接进 `/loop`**(保住无人值守 + 成本守卫);
   **可选**——没有强收尾模型的用户直接跳过。

## 后果

- 需要一条新转换:「已发布产物 → 失败现场」的**降级**(发布原本是原子、从无"发布后打回"这条路),
  写成 **branch2 根**现场,复活赛才会重分析而非在坏 ARA 上重写。
- 引入会话级**非确定性**:终审修订不可复现(人类代理本就如此),与确定性的门**明确分层**、互不混淆
  (术语见 `CONTEXT.md`:终审修订 ≠ 评价)。
- `passes_seal2` 的语义被细化:带 `final_review.json`(verdict=revised)的产物,其封印分是**终审修订
  前**的快照,provenance 显式记录,不再是沉默断言。
