# ARA Paper 抽取引擎缺陷审计

> **日期**: 2026-06-09
> **作者**: Claude Code (Opus 4.8) — 受 wangjiazhe 委托做体检
> **基准版本**: 非 Git 项目（审计对象为 `attn_sink/wm-papers/ai_package/` 下 13 个 ARA 1.0 数据包，快照日期 2026-06-09）
> **影响范围**: paper 抽取引擎（生成 ARA bundle 的 pipeline）；`ai_package/` 13 个 ARA 包；level2 自动评审子系统

---

## 结论（前置）

**核心结论**：这批 13 个 ARA 数据包**结构层面 100% 可用**（无缺失/空文件、JSON 合法、代码可解析、schema 一致、全部过 seal2），但抽取引擎存在 **3 类系统性缺陷**，按优先级排序：

1. **【P0｜源头能力缺失】代码定位（code_ref）几乎全军覆没** —— 13 包中 **12 包**被 blanket 标注 "No public repository — closed-source"，仅手工补的 CosmosTransfer1 有真实 repo+SHA。其中多篇（NVIDIA Cosmos 系列、DiffusionDrive、Vista、DreamerV3、WorldModels、Drive-WM）实际是公开开源的 —— 引擎在"代码检索"这一步**事实上没有执行检索，而是默认放弃**。
2. **【P1｜模态缺失】图片/图表层 0 产出** —— `evidence/figures/` 在 **13/13** 包中全空，引擎只抽文字表格、完全不抽架构图/定性可视化。
3. **【P1｜流程时序错位】自动评审与产物状态 drift** —— 至少 WorldVLA 一包，level2 评审给出"证据表 Table 2–6 缺失、只有索引"的 finding，但实测 `evidence/tables/` 有 5 张**带完整数值**的真实表 —— 评审在 bundle 填充**之前**就跑了，report 个别 finding 已过期失真。

**对引擎优化的含义**：P0 是"能力洞"（检索没做），P1 两条是"覆盖洞 + 编排时序洞"。三者都不是 schema/格式问题（格式已经很稳），而是**内容获取与流程编排**问题。

## 三、系统性缺陷（量化证据）

### P0 — code_ref 代码定位失效（12/13）

探针命中"No public repository — closed-source"字面的包：

```
12/13 标注'无公开仓库'：
  CosmosReason1, CosmosWorldFoundationModelPlatform, DiffusionDrive,
  DiffusionModelsAreRealTimeGameEngines(GameNGen), DrivingIntoTheFuture(Drive-WM),
  EnhancingPolicyLearningWithWorldActionModel, FastWAM, GAIA1,
  MasteringDiverseDomainsThroughWorldModels(DreamerV3), Vista,
  WorldModels, WorldVLA
唯一有真实 repo+SHA：CosmosTransfer1（手工补充的那一篇）
```

**问题性质**：12 篇里相当一部分**确有公开官方实现**。基于领域知识的高置信判断（**待你逐条复核，不作绝对断言**）：

| 论文 | code_ref 现状 | 实际开源情况（推断，待核） |
|---|---|---|
| DiffusionDrive | 无仓库 | hustvl/DiffusionDrive **开源** |
| Vista | 无仓库 | OpenDriveLab/Vista **开源** |
| DreamerV3 (MasteringDiverseDomains) | 无仓库 | danijar/dreamerv3 **开源** |
| WorldModels (Ha & Schmidhuber) | 无仓库 | hardmaru / worldmodels.github.io **有公开代码** |
| Drive-WM (DrivingIntoTheFuture) | 无仓库 | BraveGroup/Drive-WM **开源** |
| Cosmos-WFM / Cosmos-Reason1 | 无仓库 | NVIDIA/Cosmos **开源** |
| GAIA-1 | 无仓库 | Wayve **闭源**（标注正确） |
| GameNGen | 无仓库 | 无官方代码（标注大概率正确） |

→ blanket "无仓库" **不是 12 篇恰好都闭源**，而是引擎的代码检索步骤**根本没运行 / 直接 fallback 到默认值**。这是单点系统性失效，不是 12 个独立巧合。

**连带影响**：`src/execution/core.py` 因此全部是**重构 stub**（凭论文正文反推的最小可运行片段），而非官方代码。stub 本身是合法的 ARA 设计，但当官方代码存在却没链接时，下游用户会误把 stub 当权威实现。

### P1-a — 图片/图表层 0 产出（13/13）

```
有 figures 的包：0/13（evidence/figures/ 全空）
```

引擎只产出文字表格，架构图、定性渲染对比、流程图全部丢失。对世界模型/视频生成这类**强依赖视觉证据**的领域，缺图显著削弱 evidence 层价值。

### P1-b — 评审与产物 drift（时序错位）

```
WorldVLA: level2 finding 称'bundle 不含真实证据表 Table 2–6，只有索引'
          实测 evidence/tables/ = 5 张表，含完整数值
          （如 LIBERO 成功率表：OpenVLA 76.5 / WorldVLA-512 81.8 等真实数字）
```

评审在 bundle 未填充完成时就执行，导致 finding 描述的是**中间态**而非**终态**。这意味着：
- level2_report 的部分 finding **不能逐条当现状信**；
- 引擎的 pipeline 编排里，**评审 stage 早于 evidence 填充 stage 完成**（或并发无 barrier）。

### 附带 — 数据保真容差放行

```
带 AUDIT_FLAGS.md 的包：仅 CosmosTransfer1
  内容：[major] 证据数字 '7' 未被 skeptic 多数确认，源 MD 近似值为 '6.51'，
        疑似转写错误 —— 容差内放行（flagged 但未 block）
```

说明数据保真审计**机制存在且生效**（能抓转写错误），但容差策略会让可疑数字进入终稿。这是设计权衡，非 bug；但优化时应让 flagged 数字更醒目地回流。

## 四、paper 质量评估（基于 level2 自动评审）

> ⚠️ 重要前提：grade 来自**单模型自动评审 distilled bundle**，**不是论文同行评审，也不是对论文方法的独立验证**。下列结论作为"相对排序信号"使用，不作真值。

**grade 分布**：3 Strong Accept / 8 Accept / 2 Weak Accept，无 Reject。

**6 维度均分（5 分制，越低=系统性弱项）**：

| 维度 | 均分 | 最低 | 解读 |
|---|---|---|---|
| D4 论证连贯 | 4.85 | 4 | 高 —— 但主要反映 **ARA 重构的内部自洽**，非论文被证实 |
| D5 探索完整 | 4.77 | 3 | 高 —— 同上，trace 探索树质量好 |
| D1 证据相关 | 4.00 | 3 | 中 |
| D3 范围标定 | 3.92 | 3 | 中 —— 多篇 claim 过度泛化 |
| D2 可证伪 | 3.85 | 3 | 偏低 —— falsification 标准多用模糊"显著"无量化阈值 |
| **D6 方法严谨** | **3.38** | **2** | **最弱** —— 几乎每篇：无统计报告（无 CI/std/显著性检验）、缺消融、缺外部 baseline |

**findings 严重度合计**：critical 2 / major 26 / minor 31 / suggestion 7。

**必须警示的 2 篇（与抽取无关，是论文本身弱）**：
- **CosmosTransfer1**（均分 3.00，最弱，**2 个 critical**）：核心训练策略主张 C3（per-modality 独立训练优于 joint training）**零实证支撑** —— 引用的 E1/E3 只测独立训练模型质量，无 joint-training 对照组。**此包结论不可直接采信。**
- **CosmosReason1**（3.67）：声称异步 RL 带来 ~160% 训练效率提升，指向的 E4 **不含任何效率指标**。

> 注：D4/D5 偏高 ≠ 论文强，而是引擎重构得自洽；真正反映论文质量的是 D2/D6 偏低 —— 这是该研究方向（世界模型/具身）的共性短板，非引擎问题。

## 五、缺陷 → 引擎 stage 归因（优化靶点）

| # | 缺陷 | 严重度 | 归因 stage | 建议修复方向 |
|---|---|---|---|---|
| P0 | code_ref 12/13 默认"无仓库" | 高 | 代码检索 stage | 接入真实仓库检索（arXiv→GitHub/PapersWithCode/官方主页），检索失败与"确认闭源"两种状态分开，**禁止用'无仓库'作默认 fallback**；闭源需有证据（如作者声明）才能下此结论 |
| P1-a | figures 0/13 | 中 | 证据抽取 stage | 增加图表抽取（图 caption + 图片落 `evidence/figures/`），至少抽架构图与主结果图 |
| P1-b | 评审-产物 drift | 中 | pipeline 编排 | 评审 stage 必须在 **evidence 全部填充完成后**才跑（加 barrier）；或评审后对产物做一致性回扫 |
| 附1 | flagged 数字进终稿 | 低 | 数据保真审计 | 维持容差机制，但 flagged 数字在 PAPER.md 正文也打标记，不只藏在 AUDIT_FLAGS |
| 附2 | core.py stub 易被误当官方码 | 低 | 代码层 | stub 文件头显式声明"reconstructed, not official"；官方码存在时优先链接 file:line |

## 六、固化建议（防回归）

按 `00-principles.md §Knowledge Persistence Discipline`，把本次发现落成可执行 gate，而非纯笔记：

1. **`tools/check_ara_bundle.py`**（submit-gate）：对每个新产 bundle 断言 ——
   - code_ref 若标"无仓库"，要求 evidence 字段（不允许裸默认）；
   - `evidence/figures/` 与 `evidence/tables/` 至少一个非空；
   - level2_report 的 "missing table" 类 finding 与实际 `tables/` 数量交叉校验，drift 即 fail。
2. **抽取后冒烟**：批量产出前先跑 1 篇（已知开源论文如 DiffusionDrive）人工目检 code_ref 是否真链到官方 repo —— 对应 `verification-methodology.md §6`。
3. **复核清单**：本报告 §3 表格逐条复核 12 篇开源性，修正 code_ref。
